基于CentOS搭建网站服务

搭建Http静态服务器环境

搭建静态网站，首先需要部署环境。下面的步骤，将告诉大家如何在服务器上通过 Nginx 部署 HTTP 静态服务。

1、 安装 Nginx

在 CentOS 上，可直接使用 yum 来安装 Nginx

yum install nginx -y

安装完成后，使用 nginx 命令启动 Nginx：

nginx

此时，访问 http://116.62.197.141 可以看到 Nginx 的测试页面

2、 配置静态服务器访问路径

外网用户访问服务器的 Web 服务由 Nginx 提供，Nginx 需要配置静态资源的路径信息才能通过 url 正确访问到服务器上的静态资源。
打开 Nginx 的默认配置文件 /etc/nginx/nginx.conf ，修改 Nginx 配置，将默认的 root /usr/share/nginx/html; 修改为: root /data/www;

配置文件将 /data/www/static 作为所有静态资源请求的根路径，如访问:http://116.62.197.141/static/index.js，将会去 /data/www/static/ 目录下去查找 index.js。现在我们需要重启 Nginx 让新的配置生效，如：

nginx -s reload

重启后，现在我们应该已经可以使用我们的静态服务器了，现在让我们新建一个静态文件，查看服务是否运行正常。
首先让我们在 /data 目录 下创建 www 目录，如：

mkdir -p /data/www

3、创建第一个静态文件

在 /data/www 目录下创建我们的第一个静态文件 index.html 

<!DOCTYPE html>
<html lang="zh">
<head>
    <meta charset="UTF-8">
    <title>第一个静态文件</title>
</head>
<body>
Hello world！
</body>
</html>

现在访问 http://116.62.197.141/index.html 应该可以看到页面输出 Hello world!
到此，一个基于 Nginx 的静态服务器就搭建完成了，现在所有放在 /data/www 目录下的的静态资源都可以直接通过域名访问。

准备域名和证书

1、域名注册

购买域名

2、域名解析

域名购买完成后, 需要将域名解析到云主机上，我的主机的 IP 为：116.62.197.141,
找到控制台，可以到控制台添加解析记录

具体解析规则点击这里
域名设置解析后需要过一段时间才会生效，通过 ping 命令检查域名是否生效

ping www.yourmpdomain.com

如果 ping 命令返回的信息中含有你设置的解析的 IP 地址，说明解析成功。

3、申请 SSL 证书

首先登陆自己的阿里云账户，找到左侧菜单的安全–SSL证书服务
点击证书购买，进入页面，这个时候可以没有出现免费证书的选项，大家不用着急，看下一步
我们选择Symantec，是不是很神奇的出现了免费型DV SSL呢，没错，我们找的就是它了，选择它购买就可以。
付款成功后返回控制台，进入我的证书–我的订单页面，是不是已经有了一个证书了。别及，现在还用不了
点击补全信息进入下一步，填写自己要绑定的域名进入下一步,继续补全信息等待审核吧，只要资料没有填错，基本都会通过的，
具体申请流程点击这里

4、安装以及nginx配置

SSL证书签发完以后，需要下载证书并安装证书

  • 打开下载的文件夹有.pem证书文件以及.key私钥文件
  • 在服务器的nginx的安装目录下新建cert文件夹，将下载的证书以及私钥文件放到此文件夹下
  • 修改nginx.conf配置文件，将证书以及私钥文件分别指向cert目录下的证书以及私钥路径

  • 配置好nginx后重启nginx服务

    nginx -s reload

    即可以通过加密的服务访问网址
