关于http的常见问题

http与https的区别

  1. HTTP 标准端口是80 ，而 HTTPS 的标准端口是443
  2. 在OSI 网络模型中，HTTP工作于应用层，而HTTPS 的安全传输机制工作在传输层
  3. HTTP 无法加密，而HTTPS 对传输的数据进行加密
  4. HTTP无需证书，而HTTPS 需要CA机构wosign的颁发的SSL证书

什么是http协议无状态协议？怎么解决这种无状态协议？

  • 无状态协议对于事物没有记忆能力。缺少状态意味着如果后续处理需要前面的信息，也就是说当客户端一次http协议完成以后，客户端在发送一次http请求，http并不知道当前客户端是“老用户”
  • 可以使用cookie来解决无状态的问题，cookie就相当于一个通行证，第一次访问的时候给客户端发送一个cookie,当客户端再次来的时候，拿着cookie(通行证),那么服务器就知道这是个“老用户”

URI和URL的区别

** URI，是uniform resource identifier, 统一资源标识符， 用来唯一的标识一个资源，一般由三部分组成

  • 访问资源的命名机制
  • 存放资源的主机名
  • 资源自身的名称，由路径表示，着重强调资源。
    ** URL是uniform resource locator，统一资源定位器，他是一种具体的URI，一般由三部分别组成
  • 协议
  • 存有该资源的主机ip地址（有时也包括端口号）
  • 主机资源的具体地址。如目录和文件名等
    **[M 53 URN，uniform resource name,统一资源命名，是通过名字来标识资源，

常见的http方法

  • GET: 用于请求已经被URI识别的资源
  • POST: 用于传输信息给服务器，只要功能与get类似
  • PUT: 传输文件，报文主体中包含文件内容，保存到对应的URI位置
  • HEAD: 获得报文首部，与get方法类似。只是不返回报文主体，一般用于验证URI是否有效。
  • Delete: 删除文件，与put方法相反，删除对应URI位置文件
  • OPTIONS: 查询相应URI支持的http方法。

http请求报文与响应报文

** 请求报文包含四部分：

  • a、请求行：包含请求方法、URI、HTTP版本信息
  • b、请求首部字段
  • c、请求内容实体
  • d、空行

    响应报文包含四部分

    a、状态行：包含HTTP版本、状态码、状态码的原因短语 b、响应首部字段
    c、响应内容实体 d、空行
